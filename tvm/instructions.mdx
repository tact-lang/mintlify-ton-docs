### How to use

- Search accepts mnemonics, opcodes, stack effects, Fift aliases, and description keywords.
- Filter by category (and subcategory where available) to narrow the view.
- Sort by mnemonic, opcode, gas cost, category, or the TVM version that introduced the instruction.
- Click or press <kbd>Enter</kbd>/<kbd>Space</kbd> on any row to see operands and bytecode details.

Need more context? See the [tvm-spec repository](https://github.com/ton-community/tvm-spec) for the full JSON schema and supporting material.

#### `E4` REPEAT

- **Category**: Cont Loops
- **Since**: v0
- **Gas**: 18

Executes continuation `c` `n` times, if integer `n` is non-negative. If `n&gt;=2^31` or `n&lt;-2^31`, generates a range check exception.
Notice that a `RET` inside the code of `c` works as a `continue`, not as a `break`. One should use either alternative (experimental) loops or alternative `RETALT` (along with a `SETEXITALT` before the loop) to `break` out of a loop.

- **Stack**: n c - 

- REPEAT REPEAT E4 Cont Loops

#### `E6` UNTIL

- **Category**: Cont Loops
- **Since**: v0
- **Gas**: 18

Executes continuation `c`, then pops an integer `x` from the resulting stack. If `x` is zero, performs another iteration of this loop. The actual implementation of this primitive involves an extraordinary continuation `ec_until` with its arguments set to the body of the loop (continuation `c`) and the original current continuation `cc`. This extraordinary continuation is then saved into the savelist of `c` as `c.c0` and the modified `c` is then executed. The other loop primitives are implemented similarly with the aid of suitable extraordinary continuations.

- **Stack**: c - 

- UNTIL UNTIL E6 Cont Loops

- **Since**: v0
- **Gas**: 34

Pushes a non-empty constant dictionary `D` (as a `Cell^?`) along with its key length `0 &lt;= n &lt;= 1023`, stored as a part of the instruction. The dictionary itself is created from the first of remaining references of the current continuation. In this way, the complete `DICTPUSHCONST` instruction can be obtained by first serializing `xF4A4_`, then the non-empty dictionary itself (one `1` bit and a cell reference), and then the unsigned 10-bit integer `n` (as if by a `STU 10` instruction). An empty dictionary can be pushed by a `NEWDICT` primitive instead.

- **Stack**:  - D n
