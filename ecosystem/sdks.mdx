---
title: "SDKs"
---

There are several ways to interact with TON blockchain

- **HTTP** libraries connect through HTTP APIs to read and write to blockchain
- **ADNL** libraries connect to [liteserver](/v3/guidelines/nodes/running-nodes/liteserver-node).

SDKs might also provide some other functionality

- **Core** libraries implement standard TON data structures (Cells, slices), formats (address, mnemonic), cryptography etc.
- **Wrappers** provide high-level APIs for interacting with standard contracts (wallet, Jetton, NFT).
- **Emulator** libraries provide a small environment similar to real blockchain for testing purposes.

There are already HTTP APIs hosted ([by third parties](ecosystem/rpc/overview)). Usually they require payments, if the volume of requests is high.
You can [self-host]() an API endpoint, if you do not trust third parties, or not ready to pay them.

You can operate your own liteserver or rent one for direct use. 
Public liteservers are often unreliable due to limited resources and high demand.
However, built-in validation prevents them from serving false data.

Your application includes a dynamically loaded, externally compiled library. 
Initial synchronization is required on the first use and may take some time to complete.
