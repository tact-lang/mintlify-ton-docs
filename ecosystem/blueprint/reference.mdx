Blueprint provides a comprehensive CLI tool for TON smart contract development. This reference covers all available commands, options, and configuration constants.

## CLI Commands

### `create`

Creates a new smart contract from a template.

#### Interactive mode
```bash
npx blueprint create
```
Launches a guided prompt to create a contract step by step.

#### Non-interactive mode
```bash
npx blueprint create <CONTRACT> --type <TYPE>
```

**Parameters:**
- `<CONTRACT>` — Contract name (will be converted to snake_case for file names)
- `<TYPE>` — Template type

**Available template types:**
- `tolk-empty` — Empty Tolk contract
- `tolk-counter` — Tolk counter contract example
- `func-empty` — Empty FunC contract
- `func-counter` — FunC counter contract example
- `tact-empty` — Empty Tact contract
- `tact-counter` — Tact counter contract example

**Example:**
```bash
npx blueprint create MyNewContract --type tolk-empty
```

**Output:**
- Creates contract file in `contracts/` directory with appropriate extension
- Generates corresponding test file in `tests/` directory
- Creates deployment script in `scripts/` directory

### `build`

Compiles smart contracts into build artifacts.

#### Interactive mode
```bash
npx blueprint build
```
Displays a list of contracts to select from.

#### Non-interactive mode
```bash
npx blueprint build <CONTRACT>
npx blueprint build --all
```

**Parameters:**
- `<CONTRACT>` — Specific contract name to build
- `--all` — Build all contracts in the project

**Examples:**
```bash
npx blueprint build MyNewContract
npx blueprint build --all
```

**Output:**
- Compiled artifacts stored in `build/` directory
- `build/<CONTRACT>.compiled.json` — Serialized contract representation
- Contains `hash`, `hex`, and `fift` fields for deployment and testing

### `run`

Executes custom scripts for deployment, interaction, and other operations.

#### Interactive mode
```bash
npx blueprint run
```
Displays a list of available scripts to select from.

#### Non-interactive mode
```bash
npx blueprint run <SCRIPT> [args...]
```

**Parameters:**
- `<SCRIPT>` — Script name (without `.ts` extension)
- `[args...]` — Optional arguments passed to the script
- `--<NETWORK>` — Network selection (`mainnet`, `testnet`)
- `--<DEPLOY_METHOD>` — Deployment method (`tonconnect`, `mnemonic`)

**Network options:**
- `--mainnet` — Use TON Mainnet
- `--testnet` — Use TON Testnet
- `--custom <URL>` — Use custom network endpoint
- `--custom-version <VERSION>` — API version (`v2`, `v3`, `liteclient`)
- `--custom-type <TYPE>` — Network type (`mainnet`, `testnet`)
- `--custom-key <KEY>` — API key for custom network

**Examples:**
```bash
npx blueprint run deployCounter --mainnet --tonconnect
npx blueprint run deployCounter --testnet --mnemonic
npx blueprint run myScript arg1 arg2
```

**Custom network example:**
```bash
npx blueprint run deployContract \
  --custom https://toncenter.com/api/v2/jsonRPC \
  --custom-version v2 \
  --custom-type mainnet \
  --custom-key <YOUR_API_KEY>
```

**Requirements:**
- Scripts must be located in `scripts/` directory
- Script files must export a `run` function:
```typescript
export async function run(provider: NetworkProvider, args: string[]) {
  // Script implementation
}
```

### `test`

Runs contract tests using the Jest testing framework.

#### Basic usage
```bash
npx blueprint test
```
Runs all test files in the `tests/` directory.

#### Specific test file
```bash
npx blueprint test <TEST_FILE>
```

**Examples:**
```bash
npx blueprint test
npx blueprint test MyContract.spec.ts
```

**Test file requirements:**
- Test files should be located in `tests/` directory
- Use `.spec.ts` or `.test.ts` extension
- Tests run in Sandbox environment for fast execution
- Supports standard Jest syntax and matchers

### `verify`

Verifies deployed contracts on blockchain explorers.

#### Basic usage
```bash
npx blueprint verify
```
Interactive mode to select contract and network.

#### Non-interactive mode
```bash
npx blueprint verify <CONTRACT> --network <NETWORK>
```

**Parameters:**
- `<CONTRACT>` — Contract name to verify
- `--network <NETWORK>` — Network (`mainnet`, `testnet`)
- `--compiler-version <VERSION>` — Compiler version used for building
- `--custom <URL>` — Custom network endpoint
- `--custom-version <VERSION>` — API version (`v2`, `v3`)
- `--custom-type <TYPE>` — Network type (`mainnet`, `testnet`)
- `--custom-key <KEY>` — API key for custom network

**Examples:**
```bash
npx blueprint verify MyContract --network mainnet
npx blueprint verify MyContract --network testnet --compiler-version 0.4.4-newops.1
```

**Custom network verification:**
```bash
npx blueprint verify MyContract \
  --custom https://toncenter.com/api/v2/jsonRPC \
  --custom-version v2 \
  --custom-type mainnet \
  --custom-key <YOUR_API_KEY> \
  --compiler-version 0.4.4-newops.1
```

**Requirements:**
- Contract must be deployed and built
- Network type must be either `mainnet` or `testnet`
- Correct compiler version must be specified

### `help`

Displays help information for Blueprint CLI commands.

```bash
npx blueprint help
npx blueprint help <COMMAND>
```

**Examples:**
```bash
npx blueprint help
npx blueprint help create
npx blueprint help run
```

### `set`

Sets project configuration values.

```bash
npx blueprint set <KEY> <VALUE>
```

**Common configuration keys:**
- Network settings
- Default compiler versions
- Project preferences

### `convert`

Converts contracts between different formats or languages.

```bash
npx blueprint convert <SOURCE> <TARGET>
```

### `rename`

Renames contracts and updates all related files.

```bash
npx blueprint rename <OLD_NAME> <NEW_NAME>
```

**Updates:**
- Contract source file
- Test files
- Deployment scripts
- Import statements

### `pack`

Packages contracts for distribution or deployment.

```bash
npx blueprint pack <CONTRACT>
```

**Output:**
- Creates deployment-ready package
- Includes compiled artifacts
- Bundles dependencies

### `snapshot`

Creates snapshots of contract state for testing and debugging.

```bash
npx blueprint snapshot <CONTRACT>
```

**Features:**
- Captures current contract state
- Useful for regression testing
- Helps with debugging state changes

## Configuration

Blueprint can be configured using a `blueprint.config.ts` file in your project root. This file allows you to pre-define network parameters, plugins, and other CLI options.

### Basic configuration

```typescript
import { Config } from '@ton/blueprint';

export const config: Config = {
    // Configuration options
};
```

### Network configuration

```typescript
import { Config } from '@ton/blueprint';

export const config: Config = {
    network: {
        endpoint: 'https://toncenter.com/api/v2/jsonRPC',
        version: 'v2',
        type: 'mainnet',
        key: '<YOUR_API_KEY>',
    },
};
```

**Network options:**
- `endpoint` — Network endpoint URL
- `version` — API version (`v2`, `v3`, `liteclient`)
- `type` — Network type (`mainnet`, `testnet`)
- `key` — API key for authenticated access

### Liteclient configuration

```typescript
import { Config } from '@ton/blueprint';

export const config: Config = {
    network: {
        endpoint: 'https://ton.org/testnet-global.config.json',
        version: 'liteclient',
        type: 'testnet',
    }
};
```

### Request timeout

```typescript
import { Config } from '@ton/blueprint';

export const config: Config = {
    requestTimeout: 10000, // 10 seconds
};
```

### Recursive wrappers

```typescript
import { Config } from '@ton/blueprint';

export const config: Config = {
    recursiveWrappers: true, // Default: false
};
```

Controls whether the `wrappers` directory is searched recursively for contract configurations.

### TON Connect manifest

```typescript
import { Config } from '@ton/blueprint';

export const config: Config = {
    manifestUrl: 'https://yourdomain.com/custom-manifest.json',
};
```

Override the default TON Connect manifest URL when using TON Connect provider.

## Environment Variables

Blueprint supports environment variables for wallet configuration when using mnemonic provider:

- `WALLET_MNEMONIC` — Wallet mnemonic phrase (space-separated words)
- `WALLET_VERSION` — Wallet contract version (`v1r1`, `v1r2`, `v1r3`, `v2r1`, `v2r2`, `v3r1`, `v3r2`, `v4r1`, `v4r2`, `v4`, `v5r1`)
- `WALLET_ID` — Wallet ID (for versions below `v5r1`)
- `SUBWALLET_NUMBER` — Subwallet number (for `v5r1` wallets)

### Example .env file

```bash
WALLET_MNEMONIC="word1 word2 word3 ... word24"
WALLET_VERSION="v4"
WALLET_ID="698983191"
```

## Constants and Default Values

Based on the Blueprint CLI constants, the following default values and configurations are used:

### Template Types
- `tolk-empty`, `tolk-counter` — Tolk contract templates
- `func-empty`, `func-counter` — FunC contract templates  
- `tact-empty`, `tact-counter` — Tact contract templates

### Network Endpoints
- **Mainnet:** `https://toncenter.com/api/v2/jsonRPC`
- **Testnet:** `https://testnet.toncenter.com/api/v2/jsonRPC`
- **Liteclient Mainnet:** `https://ton.org/global.config.json`
- **Liteclient Testnet:** `https://ton.org/testnet-global.config.json`

### Default Directories
- `contracts/` — Contract source files
- `tests/` — Test files
- `scripts/` — Deployment and interaction scripts
- `build/` — Compiled artifacts
- `wrappers/` — Contract wrappers

### File Extensions
- `.tolk` — Tolk contracts
- `.fc` — FunC contracts
- `.tact` — Tact contracts
- `.spec.ts`, `.test.ts` — Test files
- `.compiled.json` — Build artifacts
