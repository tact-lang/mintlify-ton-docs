The generalized checklist for integrating any dApp with TON looks as such:

<Steps>
  <Step title="Install necessary libraries">
    Apart from the relevant TON Connect packages, you might also need general a general SDK to cover broader integration needs, like jetton (token) or NFT-related things.
  </Step>
  <Step title="Create a TON Connect manifest">
    TON Connect manifest is a small JSON file that lets wallets discover information about your dApp.
    It should correspond to a [valid definition](/reference/infrastructure/sdks/ton-connect/manifest) and be named `tonconnect-manifest.json`.

    For a web dApp, place it at `https://[myapp.com]/tonconnect-manifest.json` so that wallets can fetch it with a simple GET request (no CORS or auth).

    <Card
      title="TON Connect manifest in the Reference"
      href="/reference/infrastructure/sdks"
    />
  </Step>
  <Step title="Add a button in the UI">
    Users need a clear way of connecting their wallets to your app, so you must give a clear UI element to do so. Usually, that's a "Connect wallet" button.

    That said, when opening your web dApp from some wallets might automatically open up a wallet connection modal window 

    and they might not open it from an in-wallet browser, 
  </Step>
  <Step title="Use the wallet connection">
    At this step, everything is ready for use — you just need to add necessary logic to send messages.
  </Step>
</Steps>

## Minimal React example

<Aside type="caution">

  The example sends **real TON** from the connected wallet. Try it with a testnet wallet first or send less Toncoin (say, 0.01 instead of 1) to avoid surprises.

</Aside>

Add the `@tonconnect/ui-react` dependency.

```bash
npm i @tonconnect/ui-react
```

Then, create the following `App.tsx`:

```tsx title="App.tsx"
import { TonConnectUIProvider, ConnectButton } from '@tonconnect/ui-react';

export default function App() {
  return (
    <TonConnectUIProvider
      manifestUrl="https://myapp.com/tonconnect-manifest.json">
      <ConnectButton />
    </TonConnectUIProvider>
  );
}
```

Once the user has connected, you can prepare and send a message from the wallet:

```ts
await tonConnectUI.sendTransaction({
  validUntil: Math.floor(Date.now() / 1000) + 300,
  messages: [{
    address: wallet.account.address,
    amount: '1000000000' // 1 Toncoin, specified in nanoToncoin format (10¯⁹)
  }]
});
```

See this example in action by cloning and locally running the code in the corresponding repository:

```shell
git clone https://github.com/memearchivarius/react-ui-example.git
cd react-ui-example
npm install
npm run dev # http://localhost:5173
```

Open your browser at `http://localhost:5173`, connect a wallet and try the *Send 1 TON* button.

## Minimal HTML/JS example

<Aside type="caution">

  The example sends **real TON** from the connected wallet. Try it with a testnet wallet first or send less Toncoin (say, 0.01 instead of 1) to avoid surprises.

</Aside>

For quick testing, you can use a single-file HTML example.

```html title="index.html" expandable
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <title>TON Connect sandbox</title>
  <script src="https://unpkg.com/@tonconnect/ui@latest/dist/tonconnect-ui.min.js"></script>
</head>
<body>
  <div id="ton-connect"></div>
  <button id="send" disabled>Send 1 TON</button>
  <script>
    const ui = new TON_CONNECT_UI.TonConnectUI({
      // Let's re-use the manifest from the demo app
      manifestUrl: 'https://ton-connect.github.io/demo-dapp-with-react-ui/tonconnect-manifest.json',
      buttonRootId: 'ton-connect'
    });
    ui.onStatusChange(w => document.getElementById('send').disabled = !w);
    document.getElementById('send').onclick = async () => {
      // This will send 1 Toncoin from the connected wallet, beware!
      await ui.sendTransaction({
        validUntil: Math.floor(Date.now()/1000)+300,
        messages:[{address: ui.account.address, amount: '1000000000'}]
      });
    };
  </script>
</body>
</html>
```
